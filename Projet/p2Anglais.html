Project 2

Development of the QGChat Web Application

QGChat is a web application that allows users to create and manage discussion threads with one or more participants. Each user can post and read messages in these threads.

The project follows an MVC architecture in JEE and is designed to be responsive, ensuring smooth usage on both desktop and mobile devices.

This project allowed me to apply my knowledge in web development and relational database management.

Skills Learned

  • Development in JEE: Using Java Enterprise Edition to structure the application following the MVC model.
  • Database Management: Designing and implementing a relational database with PostgreSQL.
  • SQL Query Security: Using prepared statements to prevent SQL injections.
  • XSS Protection: Filtering user inputs with escapeHtml4.
  • Secure Authentication: Implementing an authentication system with session management and JWT security.
  • Frontend Development: Creating a responsive user interface with HTML, CSS, and JavaScript.
  • Message and Reaction Management: Implementing messaging features with Markdown support and emoji reactions.
  • Role and Permission Management: Developing a system for managing administrators and channel subscriptions.
  • Server Configuration: Setting up and managing the server via a YAML configuration file.
  • API: REST API controller.
