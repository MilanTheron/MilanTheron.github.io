Projet 2

Développement de l'application web QGChat

QGChat est une application web permettant aux utilisateurs de créer et gérer des fils de discussion avec un ou plusieurs participants. Chaque utilisateur peut poster et lire des messages dans ces fils.

Le projet suit une architecture MVC en JEE et est conçu pour être responsive, garantissant une utilisation fluide sur ordinateur et mobile.

Ce projet m'a permis de mettre en pratique mes connaissances en développement web et en gestion de bases de données relationnelles.

Compétences apprises

  • Développement en JEE : Utilisation de Java Enterprise Edition pour structurer l'application selon le modèle MVC.
  • Gestion des bases de données : Conception et implémentation d'une base de données relationnelle avec PostgreSQL.
  • Sécurité des requêtes SQL : Utilisation de requêtes préparées pour éviter les injections SQL.
  • XSS Protection : Filtrage des entrées utilisateur avec escapeHtml4.
  • Authentification sécurisée : Implémentation d'un système d'authentification avec gestion des sessions et sécurisation avec JWT.
  • Développement Frontend : Création d'une interface utilisateur responsive avec HTML, CSS et JavaScript.
  • Gestion des messages et réactions : Mise en place des fonctionnalités de messagerie avec support du Markdown et des réactions par emoji.
  • Gestion des rôles et permissions : Développement d'un système de gestion des administrateurs et abonnements aux channels.
  • Configuration du serveur : Paramétrage et gestion du serveur via un fichier de configuration YAML.
  • API : Contrôleur API REST.
